# P2: Clarify ZIP vs CLI Export Differences

**Priority**: P2 - MEDIUM
**Date**: 2025-12-24
**From**: Quality Agent

---

## Problem Statement

The Export step presents 4 options (CLI Command, Download ZIP, Pull from Platform, 
Install Wizard) but doesn't clearly explain what each provides. The downloaded 
README.md adds to the confusion:

> "For full templates with pre-built components, use the CLI command"

This implies the ZIP is NOT a full template, but users expect it to be since they:
1. Selected a template (E-commerce)
2. Saw a full preview with components
3. Went through 8 configuration steps

## Required Documentation Updates

### Update 1: Export Step UI Copy

Replace current descriptions with clear comparison:

| Option | What You Get | Best For |
|--------|--------------|----------|
| **CLI Command** | Full template with all components, pages, and integrations pre-configured | Developers who want production-ready code |
| **Download ZIP** | Starter structure with dependencies listed, .dd/ context files | Developers who want to build from scratch with guidance |
| **Pull from Platform** | Same as CLI, synced to cloud for team sharing | Teams and collaboration |
| **Install Wizard** | Guided setup with system detection | Beginners (coming soon) |
