TASK: Create Dynamic AI-Composed Templates
PRIORITY: P1 (Differentiation)
TYPE: Templates / AI / Architecture
AGENT: Template Agent

---

## Objective

Replace static pre-built templates with an AI-composed template system
that assembles the optimal structure based on user requirements.

## Current Problems

1. Limited templates (saas, ecommerce, blog, portfolio)
2. Templates are monolithic, not composable
3. Can't mix patterns from different templates
4. Template choice is binary, not gradual
5. No custom template generation

## Deliverables

### 1. Template Component Library

Break templates into composable building blocks:

```

├── patterns/

│   │   ├── hero-centered.tsx

│   │   ├── pricing-simple.tsx
│   │   ├── pricing-comparison.tsx
```

### 2. AI Template Composer

System that:
- Analyzes user's vision + research results
- Selects optimal patterns for each page
- Composes pages from pattern library
- Generates custom patterns when needed

```typescript
interface TemplateComposition {
  pages: {
    path: string;
    layout: string;
    patterns: {
      slot: string;  // "hero" | "features" | "pricing"
      pattern: string;  // "hero-split" | "hero-centered"
      reason: string;  // "User mentioned video, using hero-video"
    }[];
  }[];
}
```

### 3. Pattern Recommendation Engine

When user describes their project:
- AI suggests specific patterns with reasoning
- User can accept/reject/swap patterns
- System learns from choices (future feature)

Example:
```
User: "B2B SaaS for project management"

AI recommends:
- hero-split (shows product screenshot)
- features-grid (highlights capabilities)
- pricing-comparison (B2B needs feature comparison)
- testimonials-logos (B2B values social proof)
```

### 4. Custom Pattern Generation

When no existing pattern fits:
- AI generates new pattern code
- Pattern is saved to user's project
- Optionally contributed to library
