## Tasks

### Task 1: Create Type Definitions

Create the core type system at `website/lib/patterns/types.ts`:

```
website/lib/patterns/
├── types.ts              # Core types (ColorScheme, PatternDefinition, etc.)
├── registry.ts           # Pattern registry with getPattern, search, etc.
└── patterns/
    ├── heroes/
    │   ├── index.ts
    │   ├── centered-gradient.ts
    │   ├── split-image.ts
    │   ├── video-background.ts
    │   ├── animated-text.ts
    │   └── simple-centered.ts
    └── features/
        ├── index.ts
        ├── icon-grid-3col.ts
        ├── bento-grid.ts
        ├── alternating-rows.ts
        ├── comparison-table.ts
        └── feature-cards.ts
```

### Task 2: Create 5 Hero Patterns

Each hero pattern needs:
1. Pattern definition (metadata, variants, defaultProps)
2. React component implementation
3. Preview thumbnail (can be placeholder initially)

**Hero Patterns to Create:**
1. `hero-centered-gradient` - Inspired by Linear.app
2. `hero-split-image` - Inspired by Vercel
3. `hero-video-background` - Inspired by OpenAI
4. `hero-animated-text` - Inspired by Stripe
5. `hero-simple-centered` - Clean minimal hero

### Task 3: Create 5 Feature Patterns

**Feature Patterns to Create:**
1. `features-icon-grid` - 3-column icon grid
2. `features-bento-grid` - Bento box layout
3. `features-alternating-rows` - Left/right alternating
4. `features-comparison-table` - Feature comparison
5. `features-cards` - Card-based features

### Task 4: Create SectionRenderer Component

Create `website/components/preview/SectionRenderer.tsx` that:
- Maps pattern IDs to React components
- Merges default → variant → section props
- Supports editable mode with selection
- Handles missing patterns gracefully

### Task 5: Create DynamicPreviewRenderer Component

Create `website/components/preview/DynamicPreviewRenderer.tsx` that:
- Renders a full page from ProjectDefinition
- Applies branding as CSS variables
- Supports viewport switching (desktop/tablet/mobile)
- Supports editable mode
