# P1 Task: Fix Dark Mode Background - Remove Purple Gradient

> **Priority**: P1 - High
> **From**: Quality Agent (visual review)
> **Created**: 2025-12-25
> **Depends On**: P1-warm-neutral-color-scheme.txt (completed)
> **Status**: Ready for execution

---

## Problem

The dark mode currently uses a **purple/pink gradient background** which clashes with the new Orange/Stone brand colors:

- Purple (cool) vs Orange (warm) creates visual tension
- Complementary colors fighting for attention
- Inconsistent brand feel between light and dark modes
- Geometric overlay image adds visual noise

**Light mode looks great** - clean, warm, premium. Dark mode needs to match this aesthetic.

---

## Solution

Replace the purple/pink gradient with **warm dark stone tones** that complement the Orange accent colors.

---

## Color Specifications

### Dark Mode Background Options

| Element | Current (Wrong) | New (Correct) |
|---------|-----------------|---------------|
| Base background | Purple gradient | `stone-950` (#0C0A09) |
| Secondary surface | Pink tones | `stone-900` (#1C1917) |
| Cards/panels | Purple overlay | `stone-800` (#292524) |
| Subtle gradient | Purple → Pink | `stone-950` → `stone-900` |

### Dark Mode Text

| Element | Color |
|---------|-------|
| Headings | `stone-100` (#F5F5F4) |
| Body text | `stone-300` (#D6D3D1) |
| Muted text | `stone-400` (#A8A29E) |
| Orange accents | `orange-400` (#FB923C) or `orange-500` (#F97316) |

## Implementation Steps

### Step 1: Find and Remove Purple Gradient

Search for purple/violet/pink gradient definitions in:
- `website/app/globals.css`
- `website/tailwind.config.js`
- Any component using gradient backgrounds

Look for patterns like:
```css
/* Remove these */
background: linear-gradient(...purple...pink...);
bg-gradient-to-* from-purple-* to-pink-*
bg-violet-* bg-purple-* bg-pink-*
```

### Step 2: Replace with Warm Dark Background

```css
/* Dark mode base */
.dark {
  --background: 12 10 9; /* stone-950 */
  --foreground: 245 245 244; /* stone-100 */
}

/* Or in Tailwind classes */
dark:bg-stone-950
dark:bg-gradient-to-br dark:from-stone-950 dark:to-stone-900
```

### Step 3: Update Hero Section Background

The hero section specifically needs:

```tsx
// Before (purple gradient)
<div className="bg-gradient-to-br from-purple-900 via-pink-800 to-purple-900">

// After (warm dark stone)
<div className="bg-stone-950 dark:bg-gradient-to-br dark:from-stone-950 dark:to-stone-900">
```

### Step 4: Remove or Fix Geometric Overlay

The geometric/abstract image overlapping the terminal mockup:
- Option A: Remove it entirely (cleaner)
- Option B: Reposition it properly
- Option C: Replace with subtle pattern that matches stone tones

**Recommendation**: Remove it for cleaner aesthetic matching the light theme.

### Step 5: Verify Orange Accents Work

Ensure orange accent colors have good contrast on dark stone:
- `orange-400` (#FB923C) for links/highlights
- `orange-500` (#F97316) for buttons
- `orange-600` (#EA580C) for hover states
