# Fix Export Missing Files (P1)

## Summary
Export tests T09, T10, T14 are failing due to import path mismatches between 
generated code and exported file locations.

## Issue 1: T09 & T10 - Supabase Import Path Mismatch

**Problem**: Components import `lib/supabase.ts` but files export to `lib/supabase/` directory.

**Files that need updating**:
- `templates/blog/` components that import supabase
- `templates/dashboard/` components that import supabase
- Any login/auth pages that import from `lib/supabase`

**Fix Options**:
A) Update all imports to use `lib/supabase/client` or `lib/supabase/server`
B) Add a re-export file `lib/supabase.ts` that re-exports from `lib/supabase/index.ts`

**Recommended**: Option A - Update imports to be explicit about client vs server

## Issue 2: T14 - UploadThing Import Path Mismatch

**Problem**: Code imports `components/upload-button.tsx` but file is at 
`components/storage/upload-button.tsx`.

**Current INTEGRATION_PATHS**:
```
"storage:uploadthing": [
  ...
  "integrations/storage/uploadthing/components/storage/upload-button.tsx",
  ...
]
```

This exports to `components/storage/upload-button.tsx`.

**Fix Options**:
A) Update imports in generated code to use `components/storage/upload-button`
B) Change INTEGRATION_PATHS to place file at `components/upload-button.tsx`

**Recommended**: Option A - Keep organized `components/storage/` structure

## Issue 3: T19 - SEO Directory Structure

**Location**: `templates/seo-directory/`

**Fix Options**:
