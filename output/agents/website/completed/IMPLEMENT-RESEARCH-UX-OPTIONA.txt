# Implement Research Section UX Improvement - Option A

**Priority**: P2 - UX Improvement
**Estimated Time**: 5 minutes
**Created By**: Auditor Agent
**Date**: 2025-12-26

---

## Problem

Users are confused why Step 2 (Research) asks about "domain" when they just selected a template in Step 1. The placeholder text uses template names ("e.g., E-commerce, SaaS...") making it seem redundant.

---

## Task: Implement Option A (Quick Fix)

### File to Modify

`website/app/components/configurator/sections/ResearchSection.tsx`

### Changes Required

#### Change 1: Update the Label (line ~48)

```tsx
// FROM:
<Label htmlFor="domain" className="text-xs font-medium text-white/70">
  What domain is your project in?
</Label>

// TO:
<Label htmlFor="domain" className="text-xs font-medium text-white/70">
  Describe your specific niche
</Label>
```

#### Change 2: Update the Placeholder (line ~55)

```tsx
// FROM:
placeholder="e.g., E-commerce, SaaS..."

// TO:
placeholder="e.g., Pet food subscription, HR software for restaurants..."
```

#### Change 3: Add Hint Text (after the Input, around line ~58)

Add this line after the closing `/>` of the Input component:

```tsx
<p className="text-[10px] text-white/40 mt-1">
  Be specific! Better descriptions = better AI recommendations
</p>
```

---

## Verification

After making changes:

1. Run the dev server: `cd website && npm run dev`

4. Verify:
